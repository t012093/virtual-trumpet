# 物理モデル仕様

## 1. 基本方針

トランペットを次の結合系として実装する。

```text
口腔圧
  -> 唇の非線形バルブ
  -> マウスピース
  -> 管体とバルブ
  -> ベル
  -> 外部音圧

管体から戻る圧力
  -> マウスピース
  -> 唇へフィードバック
```

初期版では計算量と安定性を優先し、各要素を必要十分な複雑さで実装する。

## 2. 唇モデル

### 2.1 運動方程式

1質点の質量・バネ・減衰系を採用する。

```text
m * x'' + r * x' + k * (x - x0) = A * (pMouth - pCup)
```

主な状態とパラメータ:

- `x`: 唇の変位
- `x0`: 初期開口
- `m`: 有効振動質量
- `r`: 減衰
- `k`: 剛性
- `pMouth`: 口腔圧
- `pCup`: マウスピースカップ圧
- `A`: 圧力が作用する有効面積

### 2.2 非線形流量式

開口面積 `S = max(0, w * x)` （`w`: 有効唇幅）を用いて、次式で流量を計算する。

```text
符号付き流量 U を次で定義する：

Δp = pMouth - pCup

開口時 (S > 0):
  U = sgn(Δp) * S * sqrt(2 * |Δp| / ρ)

閉口時 (S <= 0 または接触中):
  U = 0

ここで ρ は空気密度（約 1.2 kg/m³）。
```

**低圧時の処理**: `|Δp|` が微小な領域では平方根の傾きが無限大となり数値的不安定を起こす。閾値 `pThreshold` を設け、`|Δp| < pThreshold` では線形領域へ滑らかに接続する。

```text
|Δp| >= pThreshold:
  U = sgn(Δp) * S * sqrt(2 * |Δp| / ρ)

|Δp| < pThreshold:
  U = sgn(Δp) * S * sqrt(2 / ρ) * (|Δp| / sqrt(pThreshold))
```

`pThreshold` の目安は 50〜200 Pa。

**逆流の扱い**: `Δp < 0` のとき `U` は負となり、管体から口腔への逆流を表す。物理的に妥当であり、この符号をそのまま管体モデルへの流入量として使う。

**唇接触時の処理**: `x <= 0` で唇が閉じた場合、流量は 0 とする。ただし数値的不連続を避けるため、接触付近では追加の復元力（ペナルティ剛性）を運動方程式に加え、変位がわずかに負になるのを許容する。

**実装上の注意**:
- `sgn(x)` は分岐ではなく `x / max(|x|, ε)` で計算し分岐予測ミスを避ける
- 平方根 `sqrt` は `|Δp|` が安全範囲にあることを事前に確認する

### 2.3 数値積分方式

**基準実装**: Newmark法 または台形則（trapezoidal rule）。減衰と圧力結合を含む系で安定性が高い。

**比較対象**: Semi-implicit Euler。計算量が少なく、Newmark法との比較検証に使う。

**要件**:
- 44.1 / 48 / 96 kHz の各サンプルレートで安定領域を検証する
- 積分器を交換可能なインターフェース（`Integrator` 抽象クラス）にする
- 管内圧 `pCup` との結合は毎ステップの陰的または半陰的更新で行う

**非推奨**: Velocity Verlet。減衰項と圧力駆動項の扱いが本質的に不得意な系であり、エネルギー保存性の利点を活かせない。

### 2.4 拡張

- 音域に応じた有効質量の変更
- 上下唇を分けた2自由度モデル
- 唇接触時の非線形剛性
- 唇幅と開口面積の独立制御
- 上流型・下流型アンブシュア差

## 3. 口腔モデル

MVPでは低域共鳴器1基として実装する。

入力:

- 平均口内圧
- 口腔容積
- 舌位置に対応する開口面積
- 損失

Version 1では、音域に応じて口腔共鳴周波数を変化させる。高音域では舌位置を高くし、有効容積を小さくするマッピングを導入する。

## 4. マウスピースモデル

カップをコンプライアンス、スロートをイナータンス、損失を抵抗として扱う集中定数モデルを採用する。

公開パラメータ:

- カップ容積
- スロート径
- スロート長
- バックボア補正
- 損失

マウスピースは管体の一部に埋め込まず、独立した状態を持つ。

## 5. 管体モデル

### 5.1 推奨方式

双方向デジタルウェーブガイドを採用する。

```text
mouthpiece -> forward delay -> bell
mouthpiece <- backward delay <- bell reflection
```

必要機能:

- 分数ディレイによる連続的な実効管長
- 周波数依存損失
- 円筒管と円錐・拡管部の接続
- セクション境界の反射
- サンプルレート変更時の再初期化

### 5.2 分数ディレイの実装

**注記**: Thiran は Allpass フィルターであり、振幅は基本的に一定。問題になるのは群遅延の周波数特性（遅延誤差）であって高域減衰ではない。

**MVP候補**: 3次 Lagrange 補間。計算量が少なく、固定ディレイ長で良好な特性を示す。

**比較対象**: Thiran Allpass。遅延誤差がフラットだが、フィルター次数分の内部状態が必要。

**検証要件**:
- 固定ディレイ長での周波数応答比較
- ディレイ長変調（Pitch Bendやスラー）時のアーティファクト比較
- 両方式を切り替え可能なインターフェースにする

### 5.3 非線形伝搬

強奏時のブラス感を再現するため、Version 1以降で振幅依存の伝搬または波形急峻化を追加する。

完全なFDTDを最初から採用せず、まず軽量な非線形セクションまたは飽和を物理的に妥当な位置へ導入する。

## 6. バルブモデル

### 6.1 基本構成

3本のバルブを管路ネットワークとして扱う。

```text
Valve 1: 約2半音分の追加管
Valve 2: 約1半音分の追加管
Valve 3: 約3半音分の追加管
```

実際の長さと補正量は測定または校正データから決定する。

### 6.2 切替方式

**方式**: 音声経路をミキシングするのではなく、バルブ開口量（または散乱係数）を連続補間する。

```text
バルブ閉 → 開:
  管内の散乱行列の係数を 0→1 へ遷移させる

バルブ開 → 閉:
  散乱行列の係数を 1→0 へ遷移させる
```

**遷移時間**: 5〜15 ms を初期値とする。実際のトランペットのバルブ過渡時間（約10〜30 ms）を参考に、音響評価と演奏性から調整する。

**遷移カーブ**: 線形または raised-cosine。圧力や流量の保存則を乱さないよう、散乱行列の無損失性を遷移中も維持する方針で設計する。

補間対象は音声サンプルではなく管路パラメータであるため、クリックや位相キャンセルのリスクが低い。

## 7. ベルモデル

ベル端では次を分離する。

- 管内へ戻る反射波
- 外部へ放射される音

MVP:

- 周波数依存の反射フィルター
- 放射用ハイパス特性

Version 1:

- 実測反射関数
- 実測放射応答
- ベル種類別プリセット

## 8. オーバーサンプリング

物理モデルは非線形要素（唇の流量、強奏時の波形急峻化）でエイリアシングを生じうるが、最初から全体固定倍率でのオーバーサンプリングは避け、段階的に導入する。

| 品質レベル | 方式 | 備考 |
|---|---|---|
| Draft | 1x（オーバーサンプリングなし） | Phase 1〜2、基本動作確認用 |
| Normal | 非線形セクションのみ 2x | 唇モデルと管体非線形部に適用。ハーフバンドフィルターでデシメーション |
| High | 物理エンジン全体 2x または 4x | Phase 5以降。エイリアシング実測とCPU計測から範囲を決定 |

各レベルでのエイリアシングをスペクトログラムで評価し、必要十分な品質レベルを選択する。

## 9. 損失とノイズ

- 壁面の粘性・熱損失
- 高域ほど大きくなる減衰
- 息に含まれる微量ノイズ
- バルブ操作時の機械ノイズは任意

ノイズを音量へ単純加算せず、口腔または唇を通る励振成分として注入できる構造にする。

## 10. 数値安定性

- 状態値の有限性をデバッグビルドで検査する
- 物理パラメータに安全範囲を設定する
- パラメータ変更を平滑化する
- 接触、平方根、除算の入力範囲を保護する
- 出力前にDC除去と安全リミッターを配置する
- 発散時には該当ボイスを安全にリセットする
- 積分器の安定性を44.1/48/96 kHzで検証する（§2.3 参照）

## 11. 校正データ

最低限必要なデータ:

- 各音域の基準唇周波数
- 息圧と音量・倍音の関係
- 管体の入力インピーダンス
- ベルの反射・放射応答
- マウスピース寸法
- バルブ組合せごとの共鳴周波数

校正データはコードに直接埋め込まず、バージョン付きのInstrument Definitionとして管理する。
