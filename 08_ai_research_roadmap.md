# AI・研究拡張計画

## 1. AIの役割

AIは物理モデルを置き換えず、次の二つへ限定する。

1. MIDI履歴から演奏パラメータを推定する
2. 物理モデルで不足する微細な音色差を補正する

最初に導入するのは1とする。

## 2. 演奏制御モデル

### 入力

- MIDIノート
- Velocity
- Note Onからの経過時間
- 前後の音程差
- 音符間隔
- CC2、CC1、CC11
- Aftertouch
- Pitch Bend
- レガート状態
- 現在の息圧と唇状態

### 出力

- 口内圧
- 舌の閉鎖量
- 唇共振周波数
- 唇有効質量
- 唇開口
- 口腔共鳴周波数
- ビブラート量

### モデル条件

- 因果的
- 小型
- 制御レート200～500 Hz
- CPUでリアルタイム動作
- 出力範囲を明示できる
- 状態をリセットできる

候補:

- GRU
- Temporal Convolution Network
- 小型LSTM
- 因果Transformerは必要性が確認できた場合のみ

## 3. 学習データ

### 初期データ

決定論的なExpression Mapperから生成する。

```text
MIDI + Controller Input
  -> Rule-based Mapper
  -> Physical Parameters
```

これによりAI統合の動作確認用データを作る。

### 実演データ

- トランペット奏者の演奏音声
- MIDIまたは演奏スコア
- ブレスセンサー
- バルブ状態
- 可能なら口内圧

直接測れない唇状態は、物理モデルへの逆推定または疑似ラベルとして扱う。

## 4. 損失関数

- パラメータ回帰損失
- 時間変化の滑らかさ
- 物理的範囲違反への罰則
- 音声のマルチ解像度スペクトル損失
- 基本周波数誤差
- アタック時刻誤差

パラメータ一致だけでなく、物理モデルを通した音響結果でも評価する。

## 5. ニューラル残差補正

Version 2以降の候補。

```text
Physical Audio
  + Physical State
  -> Causal Residual Model
  -> Corrected Audio
```

補正対象:

- 強奏時の高域成分
- 弱奏時の息成分
- タンギング直後の微細な過渡
- 音域別の放射特性

禁止事項:

- 音程やタイミングを大きく変更する
- 物理モデル入力との対応を失う
- 数十ms以上の先読みを必要とする

## 6. 音色潜在空間

将来、2次元の音色マップを追加する。

- 横軸: 柔らかい～鋭い
- 縦軸: 暗い～明るい

内部ではマウスピース、ベル、損失、唇、残差補正モデルを連続的に補間する。

## 7. 評価

- AI有効時の演奏自然性
- 操作に対する追従性
- 物理パラメータの妥当性
- レイテンシ
- CPU負荷
- 未知のフレーズへの汎化
- AI無効時との連続切替
