# 実装フローとロードマップ

## Phase 0: 開発環境

成果物:

- JUCE/CMakeプロジェクト
- AU、VST3、Standaloneの空プラグイン
- 自動ビルド
- 基本テスト

完了条件:

- GarageBandとAbleton Liveで空の音源として認識される
- MIDIイベントをログではなく内部カウンターで確認できる
- 状態保存と再読込が動作する

**Phase 0 は即時着手可能。物理モデルの詳細決定を待たずにプラグイン基盤を構築できる。**

## Phase 1: 最小物理モデル

**Phase 1 開始前に決定すべき項目:**

- 非線形流量式の完全な定式化（§2.2 参照）
- 数値積分方式（§2.3 参照）
- 唇と管内圧の結合方式

実装:

- 1質点唇モデル（Newmark法または台形則を基準実装とする）
- 固定管長ウェーブガイド（分数ディレイは 3次 Lagrange を MVP 候補とする）
- 簡易ベル反射
- 息圧入力
- DC除去と安全リミッター

完了条件:

- 持続的な自励振動が発生する
- 息圧を下げると自然に停止する
- パラメータ全範囲で発散しない

**Phase 1 完了前に確定すべき項目:**

- 分数ディレイ方式（Lagrange vs Thiran の比較検証）
- 44.1/48/96 kHz での安定性試験
- CPUバジェット目標値の検証

## Phase 2: MIDI演奏

実装:

- MIDI Interpreter
- Note、Velocity、CC2、Pitch Bend
- モノフォニック音符優先規則
- アタック、持続、リリース
- 基本レガート

完了条件:

- 主要音域を鍵盤で演奏できる
- タンギングとレガートで異なる過渡応答が出る
- ピッチベンドでクリックが出ない

## Phase 3: 楽器構造

実装:

- マウスピース共鳴
- 可変断面管
- 3バルブ
- ベル放射
- 周波数依存損失

完了条件:

- 運指と倍音系列から音程を決定できる
- バルブ切替が安定する（散乱係数補間方式、5〜15ms 遷移）
- 音域ごとの音色差が物理モデルから生じる

**Phase 3 までに確定すべき項目:**

- バルブ遷移方式の最終決定（散乱係数補間 vs 代替方式の比較）
- オーバーサンプリング品質レベルの選定（Draft/Normal/High、§8 参照）

## Phase 4: 製品MVP

実装:

- UI
- プリセット
- MIDI Learn
- パラメータオートメーション
- macOS署名前の配布ビルド
- DAW互換性テスト

完了条件:

- AU/VST3で同じプリセットを再現できる
- DAWプロジェクト再読込時に状態が復元される
- 30分以上の連続演奏で異常が起きない

## Phase 5: 測定と校正

作業:

- 実物トランペット録音
- 入力インピーダンス資料または測定値の収集
- ベル応答の推定
- 音域、強弱、奏法別の比較

成果物:

- バージョン付きInstrument Definition
- 校正スクリプト
- 参照音声
- 自動比較レポート

## Phase 6: AI演奏支援

作業:

- 演奏入力と物理パラメータの記録
- Fallback Controllerによる教師データ作成
- 小型因果モデルの学習
- C++推論実装
- AI/手動の連続切替

完了条件:

- NoteとVelocityだけでも自然なフレーズになる
- ブレス入力がある場合は奏者の操作を優先する
- AI有効化で遅延や音切れが増えない

## Phase 7: 高度化

- 可変唇質量
- 2自由度唇モデル
- 口腔と声道の共鳴
- 強奏時の非線形伝搬
- ミュート
- MPE
- ニューラル残差音色補正

## タスクの依存関係

```text
Plugin Skeleton
  -> Core Audio Loop
  -> Lip + Bore
  -> MIDI State Machine
  -> Mouthpiece + Bell
  -> Valves
  -> Calibration
  -> AI Controller
```

## 最初の開発スプリント

1. CMakeでJUCEプラグインを生成する
2. CoreライブラリとPlugin層を分離する
3. Fractional Delayと安全処理をテストする
4. 固定管長ウェーブガイドを鳴らす
5. 唇モデルを接続して自励振動させる
6. CC2で息圧を操作する
7. GarageBandとAbleton Liveで確認する

この段階ではUIを最小限にし、音と安定性を先に検証する。
