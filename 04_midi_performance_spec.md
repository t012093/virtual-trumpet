# MIDI・演奏制御仕様

## 1. 目的

MIDIを単なる音高と音量ではなく、トランペット奏者の行動へ変換する。

```text
MIDI
  -> 運指
  -> 倍音域
  -> 息圧
  -> 唇の剛性・質量・開口
  -> 舌と口腔
```

## 2. 標準マッピング

| 入力 | 初期割り当て |
|---|---|
| Note | 運指、倍音候補、目標音程 |
| Velocity | タンギング強度、初期息圧 |
| CC2 Breath | 継続的な息圧 |
| CC1 Mod Wheel | ビブラート量 |
| CC11 Expression | 全体表現量 |
| Channel Aftertouch | 唇の締まり、明るさ |
| Pitch Bend | リップベンド |
| CC64 Sustain | レガート優先 |

ユーザーがCC割り当てを変更できるようにする。

## 3. 音程決定

MIDIノートを直接、管長へ変換しない。

```text
MIDI Note
  -> 候補運指
  -> 各運指の共鳴系列
  -> 目標に最も適した倍音番号
  -> 唇共振周波数
  -> 微調整
```

同じ音程に複数の運指がある場合は、前後の音、移動距離、設定された奏法から選択する。

## 4. 状態機械

```text
Idle
  -> PreAttack
  -> Attack
  -> Sustain
  -> LegatoTransition
  -> Release
  -> Idle
```

### タンギング

- Note On時に前の音が鳴っていない
- または音符間に一定以上の空白がある
- または再タンギング指示がある

Velocityから舌の閉鎖量、開放速度、初期息圧を決定する。

### スラー

- 前の音が継続中
- Sustain PedalまたはLegatoモードが有効
- 音符間隔が設定値以下

息を止めず、唇の基準周波数、運指、管路を連続的に移行する。

## 5. 演奏モード

### Easy

- NoteとVelocityだけで演奏可能
- 息圧、ビブラート、音域移行を自動生成
- 初心者向け

### Expressive

- CC2、Aftertouch、Pitch Bendを優先
- 自動補助は入力がない部分だけ適用

### Physical

- 入力値を物理パラメータへ直接近い形で割り当てる
- 研究とサウンドデザイン向け

## 6. パラメータ平滑化

### アルゴリズム

デフォルトの平滑化には 1次 IIR ローパス（one-pole filter）を使用する。

```text
y[n] = α * target + (1 - α) * y[n-1]
α = 1 - exp(-1 / (fs * τ))
```

ここで `fs` は制御レート（オーディオレートで駆動する場合はサンプルレート）、`τ` は時定数。

タンギング検出時（Note On から一定期間、または Velocity が閾値以上）は時定数を短縮し、速い応答を得る。具体的には `τ` を 1/5〜1/10 に縮小する。

線形ランプに切り替えたいパラメータ（例: 管長の大きな遷移）では、1次IIRに加えて最大変化率制限（slew-rate limiter）を併用する。

### 時定数目安

| パラメータ | 目安 |
|---|---:|
| 息圧 | 5～50 ms |
| 唇剛性 | 2～20 ms |
| 唇開口 | 2～20 ms |
| 管長 | 1～10 ms |
| ビブラート量 | 20～100 ms |
| 出力ゲイン | 5～30 ms |

値は奏法に応じて変える。タンギングでは速く、柔らかいスラーでは遅くする。

## 7. Pitch Bend

Pitch Bendは出力音の後段ピッチシフトではなく、次へ作用させる。

- 唇の共振周波数
- 必要に応じて実効管長
- 口腔共鳴

標準範囲は±2半音とし、±12半音まで設定可能にする。

## 8. ビブラート

ビブラートは複数成分に分ける。

- 唇共振周波数の変動
- 息圧の変動
- 微小な音量変動

単純な出力ピッチLFOだけにしない。開始直後は抑え、持続音で徐々に増加できるようにする。

## 9. MPE

Version 1以降で対応する。

- Per-note Pitch Bend
- Per-note Pressure
- Per-note Slide

基本はモノフォニックだが、MPE情報を現在の主音へ適用し、将来の複数ボイス対応を妨げないイベント表現にする。

## 10. AIとの境界

AIへ渡す入力:

- 過去数百msのノート履歴
- Velocity
- 音符間隔
- CCとPressure
- 現在の奏法状態
- 現在の物理状態の要約

AIから受ける出力:

- 息圧目標
- 舌の開放曲線
- 唇剛性と有効質量
- 口腔共鳴
- ビブラート

AI出力は安全範囲へ制限し、Fallback Controllerと連続的に切り替えられるようにする。
