# テスト・検証計画

## 1. テストレベル

### Unit Test

- Fractional Delay
- フィルター
- パラメータ平滑化
- 唇の積分器
- 運指表
- MIDI状態機械
- 状態のシリアライズ

### Integration Test

- 唇とマウスピース
- マウスピースと管体
- 管体とベル
- MIDIから物理パラメータまで
- プリセットから音声出力まで

### Plugin Test

- AU検証
- VST3検証
- DAW保存と再読込
- オートメーション
- サンプルレート変更
- バッファサイズ変更

## 2. 数値安定性

全パラメータについて次を行う。

- 最小値、最大値
- 急激な値変更
- ランダム操作
- 長時間持続
- 高速Note On/Off
- 極端なPitch Bend

失敗条件:

- NaNまたはInfinity
- 規定を超える出力
- 永続的なDC
- 音声スレッド停止
- 再現不能なクラッシュ

## 3. 音響検証

### 定常音

- 基本周波数
- 倍音振幅
- スペクトル重心
- 音圧
- 周期安定性

### 過渡音

- 発音遅延
- アタック時間
- タンギングのピーク
- スラー時の遷移時間
- リリース時間

### 強弱

- 息圧に対する音量
- 息圧に対する倍音増加
- 発音可能なしきい値
- 強奏時の安定性

## 4. 参照レンダリング

固定のMIDIシナリオを用意する。

```text
01_single_notes.mid
02_velocity_sweep.mid
03_legato_scale.mid
04_tongued_scale.mid
05_pitch_bend.mid
06_valve_changes.mid
07_long_note.mid
08_stress_random.mid
```

各ビルドでオフラインレンダリングし、次を比較する。

- 波形差
- スペクトル差
- ラウドネス差
- 基本周波数差
- 発音タイミング差

意図した音色変更では基準データを明示的に更新する。

## 5. レイテンシ

測定対象:

- MIDIイベントから出力開始まで
- CC入力から音響変化まで
- Pitch Bendから周波数変化まで
- AI有効時と無効時の差

目標:

- プラグイン固有の追加遅延を最小化
- 総演奏遅延10 ms以下を目標
- ジッター1～3 ms以下を目標

## 6. 性能

検証条件:

- 44.1、48、96 kHz
- 32、64、128、256、512、1024サンプル
- macOS Apple Silicon
- Windows x64

測定:

- 平均CPU時間
- 最大処理時間
- オーディオドロップアウト
- メモリ使用量
- 音声スレッドでの確保回数

## 7. DAW互換性

### GarageBand

- AUとして認識される
- MIDI録音と再生
- プロジェクト保存と再読込
- パラメータオートメーション
- サンプルレート変更

### Ableton Live

- AUとVST3として認識される
- MIDIクリップ再生
- Capture MIDI
- オートメーション
- Freeze/Flatten
- プロジェクト保存と再読込

### Reaper / Logic Pro

- 基本的なロードと演奏
- 状態保存
- オフラインレンダリング

## 8. 検証ツール

- JUCE UnitTest
- CTest
- pluginval
- `auval`
- JUCE AudioPluginHost
- DAW実機テスト
- 独自オフラインレンダラー

## 9. リリース判定

- 必須テストがすべて成功
- 既知のクラッシュがない
- 主要DAWで状態復元できる
- 30分のストレス演奏を通過
- 出力安全制限が機能する
- インストールとアンインストールが確認済み
